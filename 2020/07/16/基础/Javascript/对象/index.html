2.访问器属性的两个特性：get、set

3.内部属性：

​ 用[[]] 包裹， 如 [[prototype]] ，可以用Object.getPrototypeof()访问

2.属性描述符

定义：将一个属性的所有特性编码成一个对象返回

类别：

3.属性描述符的默认值

1.访问存在的属性

特性名 默认值
value 对应属性值
get 对应属性值
set undefined
writable true
enumerable true
configurable true

2.访问不存在的属性

特性名 默认值
value undefined
get undefined
set undefined
writable false
enumerable false
configurable false

4.描述符属性的使用规则

​ get，set 与 wirable，value 不能共存

5.判断对象属性

in：属性在指定对象或其原型链上，返回true

hasOwnProperty()：判断自身属性

.或[]：对象或原型链上不存在该属性，则会返回undefined

6.遍历

方法 特性
for ... in 遍历对象自身的和继承的可枚举属性(不含Symbol属性)
Object.keys(obj) 返回一个数组,包括对象自身的(不含继承的)所有可枚举属性(不含Symbol属性)
Object.getOwnPropertyNames(obj) 返回一个数组,包括对象自身的所有可枚举和不可枚举属性(不含Symbol属性)
Object.getOwnPropertySymbols(obj) 返回一个数组,包含对象自身的所有Symbol属性
Reflect.ownKeys(obj) 返回一个数组,包含对象自身的所有(不枚举、可枚举和Symbol)属性
Reflect.enumerate(obj) 返回一个Iterator对象,遍历对象自身的和继承的所有可枚举属性(不含Symbol属性)

11.Object.assign(target, …sources)

​ 继承属性不可枚举属性是不能拷贝的

​ 原始类型会被包装为对象

​ 拷贝访问器

22.手写递归

创建对象的多种方式

原始的创建对象方式会产生大量的重复代码

①工厂模式

抽象了创建具体对象的过程

②构造函数模式

③原型模式

可以让所有实例共享原型对象的属性和方法

④构造函数 + 原型

结合两种优点， 构造函数定义实例自己的属性，原型定义实例共享的属性

⑤寄生构造函数模式

除了使用构造函数中的this， 其他和工厂模式相似
